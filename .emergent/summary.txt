**PRODUCT REQUIREMENTS:**
*   **User Authentication:** Implement a full authentication system allowing users to register via email/username/password. All data (teams, games, stats) must be private and scoped to the logged-in user. Include a Forgot Password feature with security questions.
*   **Team Management:** Users can create teams with a name, logo, and a specific team color chosen from an advanced color picker.
*   **Statistics Tracking:**
    *   Individual Stats: Free Throws (Made/Missed), 2-Point Field Goals (Made/Missed), 3-Point Field Goals (Made/Missed), Assists, Offensive Rebounds, Defensive Rebounds, Turnovers, Steals, Blocks, and Fouls.
    *   Team Stats: Automatically calculate total team fouls and shooting percentages (FG%, 3P%, FT%).
    *   Game Flow Stats: Track and display Largest Lead, Lead Changes, and Times Tied.
*   **Game Interface:**
    *   A split-screen UI for live game stat entry, with the home team on the left and the away team on the right.
    *   Players must be sorted by jersey number in all views.
    *   Make/Miss options for all scoring attempts.
    *   A play-by-play log.
    *   An editable game title.
    *   Ability to edit player name/number during a game.
    *   A Reset Stats button to reset all game data to zero.
    *   A toggle to switch between a condensed and an expanded view of player cards.
*   **Output & Sharing:**
    *   A live, shareable, view-only link for games.
    *   Export a comprehensive, single-page PDF box score styled after NBA.com.
    *   An embeddable live game banner (1920x300px) that shows the latest live game data, including team names, scores, quarter, and leading scorers.
*   **Game Scheduling:** Ability to schedule games for a specific date and time.
*   **Branding:** The application should be branded as StatMoose Basketball with a custom moose icon and a black/dark theme. The browser tab title should be StatMoose BKB.
